# Change Control

Upstream drift is a potential invalidation event for local interpretations, controls, and assurance claims.

## Required response

1. capture the changed upstream state;
2. classify its authority and scope;
3. map affected local pages, decisions, controls, and tests;
4. approve an update or documented no-change rationale;
5. rerun validation;
6. preserve the evidence in the repository history.

{: .decision }
> A closed drift event must contain either a validated patch or an explicit, reviewable no-change rationale.
