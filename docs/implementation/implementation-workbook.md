# Implementation Workbook

Use one row for each material obligation or architecture dependency.

| Field | Required entry |
|---|---|
| Source identifier | Legal article, implementing act, ARF section, STS item, or rulebook |
| Interpreted obligation | Bounded statement of what must be achieved |
| Decision authority | Role or body authorised to decide |
| Implementing component | Service, interface, policy engine, process, or control |
| Test | Repeatable verification procedure |
| Evidence | Durable output proving the result |
| Revocation trigger | Event that invalidates or supersedes the decision |
| Owner and due date | Accountable delivery owner |

{: .decision }
> Reject rows that identify a requirement but cannot identify the authorised decision owner or testable enforcement point.
