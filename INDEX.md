## 📈 How to Use This Pack

### For Individual Onboarding
1. **All people:** Read [README.md](./README.md) + [docs/arf-explained.md](./docs/arf-explained.md)
2. **Your role:** Read role-specific reading path (40–90 min)
3. **Ongoing:** Bookmark [docs/quick-reference.md](./docs/quick-reference.md) for lookups
4. **References:** Use [REFERENCES.md](./REFERENCES.md) for external standard/CIR citations

### For Team Implementation
1. **Week 1:** All team reads [docs/arf-explained.md](./docs/arf-explained.md) together
2. **Week 1–2:** Each person reads their role-specific path
3. **Week 2–3:** Architects do [docs/architecture-layer-map.md](./docs/architecture-layer-map.md) design exercises
4. **Week 3–4:** Security does [docs/governance-to-control-mapping.md](./docs/governance-to-control-mapping.md) + threat modeling
5. **Ongoing:** Use guides as implementation progresses; cite from [REFERENCES.md](./REFERENCES.md) in evidence docs

### For Program Offices
1. **Monthly:** Monitor [CHANGELOG.md](./CHANGELOG.md) for on-ramp updates
2. **Quarterly:** Check [docs/upstream-alignment-guide.md](./docs/upstream-alignment-guide.md) for ARF changes and CIR updates
3. **When stuck:** Share relevant reading path with team
4. **Compliance:** Use [REFERENCES.md](./REFERENCES.md) for regulatory citation in governance documents

### For Contributors
1. **Before adding content:** Check [REFERENCES.md](./REFERENCES.md) for existing citations
2. **When citing:** Follow [CONTRIBUTING.md](./CONTRIBUTING.md) citation style guide
3. **Verify links:** Test each URL before submitting PR
4. **Update REFERENCES.md:** If adding new external resources
